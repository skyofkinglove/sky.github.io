guess id

此题巨坑
题目给出了3个功能

一开始我以为问题出在注册上
在我疯狂测试后，发现身份证号和国籍如果过长，会让数据库抛出错误

于是我发现，mmp
可控的身份证号和籍贯被base64后再进行数据库操作
用户名直接只允许字母+数字，而密码是被md5后才进行操作
所以根本不存在注入问题
那么问题只能出在修改个人信息上了？
这里我也是真的服了，不知道什么原因，我们修改个人信息，竟然后台会有Bot自动点，进行查看？
我还是自己随便测试xss发现的……真的是为了出题而出题，服了
但是事情并没有这么简单
